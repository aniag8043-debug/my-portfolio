# 나의 주식 포트폴리오 — 클라우드 자동 갱신 v4

GitHub Actions가 매일 한국시간 09:00, 16:00에 자동 실행되어 포트폴리오 페이지를 갱신합니다.
노트북이 꺼져 있어도 클라우드(GitHub 서버)에서 동작하고, 휴대폰 어디서나 웹 URL로 접근 가능합니다.

## 비용

**무료.** GitHub Free 계정 한도 안. 한 번 실행 약 2~3분 소요.

## 매매 후 갱신 방법

`portfolio_data.json`에서 해당 종목의 `qty`, `cost`만 수정 → push.
다음 갱신 시각에 자동 반영됩니다. 새 종목 추가는 객체 하나 추가.

```json
{
  "name": "삼성전자",
  "account": "신규",
  "accNum": "xxx-xx-xxxxxx",
  "market": "KR",
  "category": "Stock",
  "qty": 10,
  "cost": 75000,
  "ccy": "KRW",
  "yahoo": "005930.KS",
  "fdr": "005930"
}
```

## 수동 갱신 (즉시 반영)

GitHub 리포 → **Actions** 탭 → "포트폴리오 자동 갱신" → **Run workflow** 클릭.
약 2~3분 후 페이지 새로고침하면 반영.

## 일정 변경

`.github/workflows/update.yml`의 cron 표현식 수정. GitHub Actions cron은 **UTC 기준**이라 한국시간에서 9시간 뺍니다.

| 한국시간 | UTC cron |
|---|---|
| 09:00 | `0 0 * * *` |
| 16:00 | `0 7 * * *` |
| 08:30 평일만 | `30 23 * * 0-4` |

## 알려진 한계 — 솔직히

이 자동화는 **완벽하지 않습니다**. 특히 다음 항목들은 운에 따라 일시적으로 실패할 수 있습니다.

### 1. 신규/특수 ETF — 가격 누락 가능
`0183J0` (TIGER 미국우주테크), `530055` (KB 천연가스 ETN) 등 신규 상장이나 ETN은 야후 파이낸스·FDR에 등재 안 되어 있을 수 있습니다. 그 경우 `price=null`로 표시되며, 손익도 계산되지 않습니다. 해결: 종목코드를 정확히 찾아 `portfolio_data.json` 수정.

### 2. 한국 종목 펀더멘털 — yfinance 미지원 다수
yfinance는 미국 종목의 PER/PBR/EPS는 잘 가져오지만, 한국 종목은 누락이 많습니다. 따라서 한국 종목 PER 컬럼은 대부분 "—" 표시. 정확한 펀더멘털을 자동 수집하려면 FnGuide 크롤링 또는 KRX 데이터 마켓플레이스 별도 연동 필요.

### 3. 네이버 금융 크롤링 — 구조 변경 시 영향
네이버 금융 페이지 구조가 바뀌면 한국 뉴스 수집이 실패합니다 (1~2년에 1번 정도 발생). 그때는 `fetch_news_kr()` 함수의 CSS 셀렉터(`table.type5 tbody tr` 등)를 수정해야 합니다.

### 4. 공시(DART) 미구현
"3번 뉴스/공시" 요구사항 중 **공시는 구현되지 않았습니다**. 이유: DART Open API 키를 사용자가 별도 발급받아야 하고(opendart.fss.or.kr, 무료, 5분), GitHub Secrets에 등록해야 하기 때문. 필요하시면 별도 안내드릴 수 있습니다.

### 5. 이동평균선(5/20/120/50주) ✅ v3에서 구현
yfinance/FDR 일봉 250일치를 받아 pandas rolling으로 계산. 50주선은 250영업일 평균.
단, **신규 ETF는 일봉 데이터가 250일 안 모인 경우 50주선이 "자료 부족"으로 표시됨.**

### 6. 시장 키워드 ✅ v4에서 구현
- pykrx로 거래대금/등락률 상위 추출 (KRX 공식 데이터, 매우 안정)
- 네이버 테마는 크롤링 — **네이버가 페이지 구조를 바꾸면 일시적으로 빈 상태로 표시될 수 있음**.
  여러 셀렉터 패턴을 시도하지만 그래도 실패 시 `fetch_naver_themes()` 함수 안의 셀렉터를 수정해야 함.

### 9. 업종별 외/기/개 수급 — KRX 미제공
v3에서 KOSPI 21개 업종지수의 등락률/거래대금은 추가했지만, **업종별 외국인/기관/개인 수급은 KRX가 직접 노출하지 않는 데이터**입니다. 종목 단위로 매핑해서 합산하는 방법은 가능하나 GitHub Actions에서 timeout 위험이 있어 권장하지 않습니다. 시장 단위(KOSPI/KOSDAQ) 외/기/개 수급은 "💧 시장 수급" 섹션에서 확인 가능.

### 7. M7 분석 텍스트는 헤드라인만
"전날 미국 증시 데일리 분석"의 **분석 텍스트**(데일리 브리핑, 한국증시 영향 분석)는 사람의 큐레이션이 필요해서 자동화 미구현. 현재는 가격·등락률·헤드라인 2건만 표시.

### 8. pykrx 첫 실행 약 1~2분 소요
KRX 서버 응답 속도에 따라. 종종 timeout 발생 시 재실행하면 됨.

## 데이터 소스 정리

- **yfinance**: 미국 종목 가격·펀더멘털·뉴스, 환율, 코스피
- **FinanceDataReader**: 한국 종목 가격(폴백), 코스피 지수
- **pykrx**: 시장 수급 (외국인/기관/개인 매매대금)
- **네이버 금융**: 한국 종목 뉴스 (크롤링)
